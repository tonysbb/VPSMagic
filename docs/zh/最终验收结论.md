# 当前版本最终验收结论

## 结论

当前版本已经从“联调脚本集合”收口成一套可以交给陌生用户使用的恢复工具原型。

它不是对所有环境都承诺一键完整恢复的系统镜像工具，但已经具备：

- 明确的新用户入口
- 可重复的本地 / 远端恢复主链路
- 清楚的前置检查与失败提示
- 配置级快照与可审计边界
- 中文与英文两套可用的主流程体验

## 已完成验收的主链路

### 中文链路

- `init`
- `doctor`
- `restore`
- `backup`

### 英文链路

- `init`
- `doctor`
- `restore`
- `backup`

### 真实空机远端恢复链路

已用真实空机目标机完成以下验证：

1. `init` 可生成远端恢复配置
2. `doctor` 会按配置和语言输出接入建议
3. `restore` 会执行远端前置检查并过滤不可用目标
4. `OCI` 不可用时可以回退到其他已配置远端
5. Debian 空机缺少 `docker-compose-plugin` 时，会继续尝试 Docker 官方仓库
6. 最终恢复结果通过健康检查，摘要为 `警告: 0`、`错误: 0`

## 已验证通过的关键能力

### 1. 新用户 onboarding

- 首次 `init` 可交互选择界面语言
- 不再要求用户先理解 `OCI / R2 / rclone`
- 支持：
  - 仅本地模式
  - 本地 + 远端模式
  - 仅生成配置

### 2. 接入识别

- `doctor` 可识别：
  - Compose
  - Standalone Docker
  - Systemd
  - Reverse proxy
  - Database
  - User home
- 支持读取 `--config`
- 支持按 `UI_LANG` 输出
- 能根据当前配置给出更贴近实际的建议路径
- 现在会额外输出恢复前风险评估，包括：
  - 当前建议
  - 风险等级
  - 阻塞项
  - 注意项

### 3. 远端恢复前置检查

- 会明确展示哪些远端可用，哪些远端因前置条件失败被排除
- `OCI` 缺少 `/root/.oci/config` 时会明确提示
- 不再把“远端不可达”误判成“备份不存在”

### 4. 远端回退能力

- 主目标不可用时可回退到备用远端
- 已真实验证：
  - `OCI -> R2`

### 5. 空机依赖自动补齐

- 缺 `rclone` 时可尝试自动安装
- 缺 Docker / Compose 时可尝试自动安装
- Debian 默认仓库只有 `docker.io`、没有 `docker-compose-plugin` 时：
  - 会继续尝试 Docker 官方仓库
- 该逻辑已在真实空机恢复中跑通

### 6. Caddy / TLS 恢复

- 反代配置恢复可用
- 针对真实复现出的 `Caddy` ACME 状态异常：
  - `Unable to validate JWS`
  - `caddy_legacy_user_removed`
- 恢复脚本已加入一次性、受条件约束的 TLS 状态自愈

### 7. 配置级快照与回滚边界

- 恢复前会生成配置级快照
- 快照目录包含：
  - `rollback_scope.txt`
  - `included_paths.txt`
  - `meta.env`
- 当前回滚边界可审计

### 8. 摘要与多语言输出

- 中文和英文主流程日志已基本一致
- 英文 `backup` / `restore` 摘要已收口
- `Docker Compose` 摘要现在显示：
  - 项目数
  - 服务数
- 当前已支持结构化输出：
  - `doctor --format json`
  - `status --format json`

## 当前明确成立的产品边界

### 1. `rclone.conf` 是安全验证门槛

工具不会自动生成远端访问配置。

这不是缺陷，而是刻意保留的安全边界。远端恢复前，目标机仍应由使用者主动准备：

- `/root/.config/rclone/rclone.conf`

### 2. `/root/.oci/config` 是 `OCI` 主目标的安全验证门槛

工具不会自动生成或复制：

- `/root/.oci/config`
- 对应私钥材料

如果目标机没有 `OCI` 凭据：

- 工具会明确提示
- 若其他远端可用，会回退到备选远端
- 无法满足前置条件时，仍可改走 `restore --local`

### 3. 轻量回滚仍然只是配置级

当前版本不回滚：

- 卷数据
- 数据库结果
- 业务副作用

## 当前不应过度承诺的部分

1. 不是所有陌生环境都保证一键完整恢复
2. 不是事务级、业务级回滚工具
3. 不自动生成远端凭据

## 当前总体判断

按成熟度看：

- 本地备份 / 本地恢复演练：已可交付
- 远端备份 / 远端恢复：已可交付，但依赖安全验证门槛
- 真实空机跨机恢复：已具备可重复验收基础
- 复杂业务的完整自动恢复承诺：仍需保守
