# 状态说明

## 标准状态视图

```bash
bash vpsmagic.sh status --config /opt/vpsmagic/config.env
```

适合用来快速查看：

- 当前机器基础信息
- 当前生效配置
- 当前启用模块
- 本地备份是否存在
- 远端备份是否可见
- 定时任务是否已安装

## 结构化输出

```bash
bash vpsmagic.sh status --config /opt/vpsmagic/config.env --format json
```

适合把结果交给：

- Shell 脚本
- CI
- 面板
- 你自己的二次封装

## 当前 JSON 字段

### `command`

固定为 `status`。

### `format`

使用 `--format json` 时固定为 `json`。

### `language`

当前实际生效的界面语言，已经综合了命令行参数和配置文件。

### `system`

当前机器身份和基础运行信息：

- `hostname`
- `operating_system`
- `kernel`
- `primary_ip`

### `configuration`

当前与备份/恢复状态相关的生效配置：

- `mode`
  - 默认是 `backup`
  - 当 `RESTORE_SOURCE_HOSTNAME` 生效时会显示为 `restore`
- `backup_root`
- `backup_destination`
- `ui_language`
- `remote_targets`
- `primary_target`
- `async_target`
- `restore_source_hostname`
- `interactive_remote_selection`
- `auto_rollback_on_failure`
- `keep_local`
- `keep_remote`
- `encryption_enabled`
- `notifications_enabled`
- `log_file`

### `enabled_modules`

当前启用的备份模块数组，例如：

- `DOCKER_COMPOSE`
- `SYSTEMD`
- `REVERSE_PROXY`

### `local_backups`

本地归档目录摘要：

- `archive_dir`
- `backup_count`
- `latest_backup`
- `latest_size_bytes`
- `total_size_bytes`

### `remote_backups`

按远端目标统计的备份可见性结果。

每一项当前包含：

- `target`
- `backup_count`
- `available`

当某个远端目标存在于配置里、但当前机器无法列出它时，`available` 会是 `false`。

如果当前没有 `rclone`，或者没有可用远端目标，这个数组可能为空。

### `schedule`

当前调度状态：

- `enabled`

## 如何理解这些结果

如果：

- `local_backups.backup_count` 为 `0`
- `remote_backups` 为空
- `schedule.enabled` 为 `false`

那通常说明这台机器虽然已经配置好，但还没有进入稳定备份节奏。

如果：

- `remote_targets` 非空
- `primary_target` 为空

通常说明目标列表存在，但配置里没有显式固定主目标。

## 和 `doctor` 的分工

建议这样区分：

- `doctor --format json` 用于恢复前判断与风险评估
- `status --format json` 用于读取当前状态
