# 恢复说明

## 先决定你要走哪种恢复方式

### 方式 A：本地恢复

适合：

- 你手里已经有 `.tar.gz` 归档
- 目标机没有远端凭据
- 你不想先处理 `rclone` / `OCI` / `R2`

### 方式 B：远端恢复

适合：

- 目标机已经具备远端访问条件
- 你想直接从远端拉取备份
- 你希望跨机恢复时少做手工传文件

这里的“远端访问条件”至少包括：

- 已准备 `rclone.conf`
- 如果主目标依赖 `OCI`，已准备 `/root/.oci/config`

这两项建议被视为安全验证门槛，而不是普通可选项。

## 远端恢复

```bash
bash vpsmagic.sh restore --config /opt/vpsmagic/config.env
```

适合你已经准备好了远端凭据和远端路径的情况。

## 本地恢复

```bash
bash vpsmagic.sh restore --local /path/to/backup.tar.gz --config /opt/vpsmagic/config.env
```

如果你只是第一次恢复，通常应优先选这一种。

## 跨机恢复

```bash
bash vpsmagic.sh restore \
  --config /opt/vpsmagic/config.env \
  --source-hostname SOURCE_HOSTNAME
```

## 当前恢复行为

- 默认先查本地备份
- 本地有备份时，可输入 `0` 切换到云端
- 云端恢复前会先做远端前置检查
- 只展示当前满足前置条件的远端
- Debian 空机默认源缺 `docker-compose-plugin` 时，会继续尝试 Docker 官方仓库
- 恢复前必须输入精确 `yes`
- 恢复前会创建配置级快照

## 第一次恢复建议怎么做

建议顺序：

1. 先做本地恢复
2. 确认摘要和健康检查合理
3. 再做远端恢复
4. 最后再做跨机恢复

这样做的原因很简单：

- 本地恢复的问题最少
- 远端恢复会额外引入凭据、网络、远端配置变量
- 跨机恢复会再额外引入主机名展开和目标机环境差异

## 恢复前建议你先确认

1. 目标机 SSH 入口正常
2. 目标机磁盘空间够用
3. 如果走远端恢复，目标机确实具备对应远端凭据，并且你明确接受这一步作为安全验证门槛
4. 你知道这次恢复是本地恢复、远端恢复，还是跨机恢复

## 恢复后建议你重点检查

1. 摘要里 `错误: 0`
2. 如果是 Compose：
   - 容器数量是否正确
   - 关键端口是否监听
   - `Compose 出网` 是否为 `ok`
3. 如果有反向代理：
   - `caddy` / `nginx` 是否 `active`
   - `80/443` 是否监听
4. 如果有单实例服务：
   - 是否按预期“恢复但不自动启动”

如果你恢复后“脚本成功但网站打不开”，不要马上断定恢复失败。先分层看：

1. 服务有没有起来
2. 反代有没有起来
3. 端口是否监听
4. 证书是否签发完成
5. DNS / Cloudflare 是否指向正确

## 无人值守恢复

```bash
bash vpsmagic.sh restore \
  --config /opt/vpsmagic/config.env \
  --auto-confirm \
  --rollback-on-failure
```

注意：

- 这里的回滚是**配置级轻量回滚**
- 不是完整系统回滚
- 不回滚卷数据、数据库结果或业务副作用
