# 快速开始

如果你是第一次使用，建议直接先运行：

```bash
bash vpsmagic.sh init
```

向导进入后，第一步会先让你选择界面语言。

当前 `init` 已支持 3 种入口：

1. `仅本地备份（推荐新手）`
2. `本地 + 云端备份`
3. `仅生成配置，稍后再完善`

对大多数第一次使用的用户，建议先选 `1`，跑通本地备份和本地恢复后，再进入云端备份。

在正式备份或恢复前，也建议先运行一次：

```bash
bash vpsmagic.sh doctor --config /opt/vpsmagic/config.env
```

现在 `doctor` 不只是识别画像，还会给出：

- 当前建议
- 风险等级
- 阻塞项
- 注意项

如果 `doctor` 已经明确提示存在“阻塞项”，不要直接进入正式恢复，先补齐条件。

## 最短成功路径

如果你不想一开始就看很多文档，按下面 4 步走就够了：

1. `bash vpsmagic.sh init`
2. 选 `1) 仅本地备份`
3. `bash vpsmagic.sh backup --config /opt/vpsmagic/config.env`
4. `bash vpsmagic.sh restore --local /path/to/backup.tar.gz --config /opt/vpsmagic/config.env`

先把这 4 步跑通，再决定要不要做远端。

## 你应该选哪条路径

### 路径 A：先只做本地备份

适合：

- 你还没有云存储
- 你还没装 `rclone`
- 你想先验证这个工具到底能不能用

建议先看：[零配置起步](./零配置起步.md)

### 路径 B：本地 + 云端备份

适合：

- 你已经确定要做异地备份
- 你接受先配置一个 `rclone` remote
- 你想把备份放到 WebDAV / S3 / 网盘

这时再继续看本页和 [配置说明](./配置说明.md)

如果你还没把“本地备份 + 本地恢复演练”做通，不建议直接跳这条路径。

## 安装

```bash
git clone https://github.com/tonysbb/VPSMagic.git /opt/vpsmagic
cd /opt/vpsmagic
bash install.sh
```

## 初始化配置（推荐）

```bash
bash vpsmagic.sh init
```

如果你选择：

- `仅本地备份`
  - 不需要先安装 `rclone`
  - 会生成 `BACKUP_DESTINATION="local"` 的配置
- `本地 + 云端备份`
  - 会进一步问你远端类型
  - 再生成远端相关配置

当前远端类型引导分为：

1. `WebDAV / OpenList / AList`
2. `S3 兼容对象存储`
3. `Google Drive / OneDrive`
4. `我已经有完整 rclone 路径，直接输入`

## 手动编辑配置（可选）

如果你不想走向导，也可以手动：

```bash
cp config.example.env config.env
vim config.env
```

## 执行一次备份

```bash
bash vpsmagic.sh backup --config /opt/vpsmagic/config.env
```

第一次跑完后，建议立刻检查：

```bash
ls -lh /opt/vpsmagic/backups/archives
sha256sum -c /opt/vpsmagic/backups/archives/*.sha256
```

如果你现在只是第一次接触这个工具，这一步做完就已经值得先停下来确认结果，不要立刻跳去远端恢复。

## 执行一次恢复

```bash
bash vpsmagic.sh restore --config /opt/vpsmagic/config.env
```

如果你当前只有本地归档，更稳的做法是：

```bash
bash vpsmagic.sh restore --local /path/to/backup.tar.gz --config /opt/vpsmagic/config.env
```

判断这轮“快速开始”是否成功，至少看这几件事：

- 备份摘要里 `错误: 0`
- 归档和 `.sha256` 都存在
- 恢复摘要里 `错误: 0`
- 关键服务状态和端口结果合理
- `doctor` 没有提示你忽略明显阻塞项直接继续

## 跨机恢复

当远端路径使用 `{hostname}` 时，在目标机执行：

```bash
bash vpsmagic.sh restore \
  --config /opt/vpsmagic/config.env \
  --source-hostname SOURCE_HOSTNAME
```

## 什么时候再配置 rclone

建议顺序：

1. 先跑通本地备份
2. 先跑通本地恢复演练
3. 再安装 `rclone`
4. 再配置云端远端
5. 最后测试远端备份 / 远端恢复

如果你直接反过来做，第一次就上云端恢复，排错成本通常会更高。
