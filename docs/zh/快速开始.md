# 快速开始

如果你已经了解基本流程，这里按场景汇总常用命令。

如果你是第一次使用，建议先看 [零配置起步](./零配置起步.md)。

## 最短成功路径

```bash
# 1. 安装
git clone https://github.com/tonysbb/VPSMagic.git /opt/vpsmagic
cd /opt/vpsmagic && bash install.sh

# 2. 初始化配置
vpsmagic init

# 3. 环境诊断
vpsmagic doctor

# 4. 执行备份
vpsmagic backup

# 5. 校验归档
sha256sum -c /opt/vpsmagic/backups/archives/*.sha256

# 6. 恢复演练
vpsmagic restore --local /path/to/backup.tar.gz
```

## 按场景速查

### 备份

```bash
# 全量备份（按 config.env 配置的目标）
vpsmagic backup

# 仅本地备份
vpsmagic backup --dest local

# 指定远端目标（临时覆盖配置）
vpsmagic backup --remote myremote:bucket/path

# 模拟运行（不实际执行）
vpsmagic backup --dry-run
```

### 恢复

```bash
# 从远端恢复（推荐）
vpsmagic restore

# 从本地归档恢复
vpsmagic restore --local /path/to/backup.tar.gz

# 跨机恢复（远端路径含 {hostname}）
vpsmagic restore --source-hostname SOURCE_HOSTNAME

# 无人值守恢复 + 失败自动回滚
vpsmagic restore --auto-confirm --rollback-on-failure
```

### 在线迁移

```bash
# 直推迁移
vpsmagic migrate root@new-vps

# 指定 SSH 端口和带宽限制
vpsmagic migrate root@new-vps -p 2222 --bwlimit 10m

# 只传输备份，不执行远端恢复
vpsmagic migrate root@new-vps --skip-restore
```

### 定时备份

```bash
# 安装定时任务（默认每天凌晨 3 点）
vpsmagic schedule install

# 查看定时任务状态
vpsmagic schedule status

# 移除定时任务
vpsmagic schedule remove
```

### 状态与诊断

```bash
# 系统和备份状态概览
vpsmagic status

# 环境诊断与恢复建议
vpsmagic doctor

# JSON 格式输出（适合脚本/CI）
vpsmagic status --format json
vpsmagic doctor --format json
```

## 成功标准

每一步执行后，这样判断是否成功：

| 操作 | 判断标准 |
|------|---------|
| 备份 | 摘要 `错误: 0`，归档目录有 `*.tar.gz` + `*.tar.gz.sha256` |
| 校验 | `sha256sum -c` 输出 `OK` |
| 恢复 | 摘要 `错误: 0`，关键服务 running，端口正常监听 |
| 迁移 | 恢复摘要正常，目标机业务可访问 |
| 定时 | `vpsmagic schedule status` 显示已安装 |
