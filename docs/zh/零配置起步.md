# 零配置起步

这篇文档带你从零完成一次完整的 **备份 → 校验 → 恢复** 闭环。

## 你需要什么

- 一台运行中的 VPS（Debian / Ubuntu / CentOS 等主流 Linux 发行版）
- root 权限
- 约 10 分钟时间

**暂时不需要**：rclone、云存储、OCI 凭据。这些可以在第一次成功后再配。

## 第一步：安装

```bash
git clone https://github.com/tonysbb/VPSMagic.git /opt/vpsmagic
cd /opt/vpsmagic
bash install.sh
```

安装脚本会自动检查基础依赖（bash / curl / tar / gzip），并询问是否安装 rclone、rsync、Docker。

> 💡 如果你只是先体验，全部选 `y` 即可。如果你的 VPS 不使用 Docker，可以跳过 Docker 安装。

安装完成后，你可以在任意位置直接使用 `vpsmagic` 命令。

## 第二步：初始化配置

```bash
vpsmagic init
```

向导会先让你选择界面语言（中文 / English），然后选择备份模式：

| 选项 | 说明 | 建议 |
|------|------|------|
| `1) 仅本地备份` | 备份到本机磁盘 | 适合先体验 |
| `2) 本地 + 云端备份` | 同时备份到本机和云端 | **推荐生产使用** |
| `3) 仅生成配置` | 生成模板，稍后手动编辑 | 高级用户 |

选择 `1` 后，向导会生成 `config.env`，核心配置如下：

```bash
BACKUP_DESTINATION="local"
BACKUP_ROOT="/opt/vpsmagic/backups"
```

如果你选择 `2`，向导会继续引导你选择远端类型（WebDAV / S3 / Google Drive 等）并配置对应路径。

## 第三步：环境诊断（可选但推荐）

```bash
vpsmagic doctor
```

`doctor` 会告诉你：

- 当前机器的业务画像（Compose 应用型 / Systemd 服务型 / 轻量通用型…）
- 检测到哪些服务需要备份
- 是否存在会阻塞恢复的问题
- 风险等级和建议

如果 `doctor` 提示有"阻塞项"，先处理再继续。

## 第四步：执行备份

```bash
vpsmagic backup
```

正常完成后，你会看到类似这样的摘要：

```
╔══════════════════════════════════╗
║     备份摘要                      ║
╠══════════════════════════════════╣
  总模块: 8
  成功: 8
  跳过: 0
  错误: 0
  归档: /opt/vpsmagic/backups/archives/vpsmagic_hostname_20260331_030000.tar.gz
```

**如何判断这一步成功：**

- 摘要里 `错误: 0`
- 归档目录同时有 `*.tar.gz` 和 `*.tar.gz.sha256` 两个文件

如果没有看到这两个文件，说明备份未完成，不要继续往下走。

## 第五步：校验备份

```bash
ls -lh /opt/vpsmagic/backups/archives/
sha256sum -c /opt/vpsmagic/backups/archives/*.sha256
```

预期输出：

```
vpsmagic_hostname_20260331_030000.tar.gz: OK
```

如果显示 `FAILED`，优先检查磁盘空间和归档完整性。

## 第六步：恢复演练

> ⚠️ **建议不要等到出故障才第一次恢复。** 先在测试环境或目标机上演练一次。

### 本地恢复（手里有归档文件）

```bash
vpsmagic restore --local /opt/vpsmagic/backups/archives/vpsmagic_hostname_20260331_030000.tar.gz
```

### 远端恢复（目标机有云端访问权限）

```bash
vpsmagic restore
```

恢复完成后，重点检查：

1. 摘要里 `错误: 0`
2. Docker Compose 容器是否全部 running
3. 反代（Caddy / Nginx）是否 active
4. 关键端口是否监听（80 / 443 等）

如果恢复摘要正常但网站打不开，按这个顺序排查：服务 → 反代 → 端口 → 证书 → DNS。

## 下一步：配置云端备份

完成本地闭环后，建议立即配置云端备份（真正的灾备能力）：

1. 确认 `rclone` 已安装（安装脚本会自动处理）
2. 配置远端存储：`rclone config`
3. 重新运行 `vpsmagic init`，选择 `2) 本地 + 云端备份`
4. 执行一次带远端的备份：`vpsmagic backup`
5. 确认远端目录里有对应的归档

远端类型选择建议：

| 类型 | 说明 |
|------|------|
| WebDAV / OpenList / AList | 最容易配置，适合起步 |
| S3 兼容存储（含 OCI、R2） | 适合已有对象存储的用户 |
| Google Drive / OneDrive | 不想折腾对象存储时使用 |

详见：[配置说明](./配置说明.md)

## 下一步：设置定时备份

```bash
vpsmagic schedule install
```

默认每天凌晨 3 点自动备份。可以通过 `config.env` 中的 `SCHEDULE_CRON` 修改。

建议同时配置 Telegram 通知，避免备份失败无人知晓。详见：[定时任务说明](./定时任务说明.md)

## 完整学习路径

```
安装 → init → doctor → backup → 校验 → restore 演练
  ↓
配置云端 → 远端备份 → 远端恢复测试
  ↓
定时备份 → Telegram 通知
  ↓
新机恢复 / 在线迁移
```
