# 零配置起步

这篇文档面向第一次使用本工具的用户，假设你当前：

- 还没有 `rclone`
- 没有 OCI / R2 / WebDAV 等远端存储
- 只想先成功做出一份可恢复的本地备份

目标很简单：

1. 在当前 VPS 上生成本地归档
2. 校验归档
3. 知道如何用这个归档做本地恢复
4. 知道什么结果算“这轮已经成功”

如果你现在只是“先想有一份能拿在手里的备份”，这篇就够了。

## 第一步：安装

```bash
git clone https://github.com/tonysbb/VPSMagic.git /opt/vpsmagic
cd /opt/vpsmagic
bash install.sh
```

## 第二步：创建最小配置

推荐直接运行：

```bash
bash vpsmagic.sh init
```

然后在向导里：

1. 配置文件路径用默认
2. 选择 `1) 仅本地备份（推荐新手）`
3. 其余大多数问题先用默认值

如果你不想走向导，再手动执行：

```bash
cp /opt/vpsmagic/config.example.env /opt/vpsmagic/config.env
vim /opt/vpsmagic/config.env
```

至少确认这两项：

```bash
BACKUP_DESTINATION="local"
BACKUP_ROOT="/opt/vpsmagic/backups"
```

说明：

- `local` 表示本次只做本地备份，不依赖远端存储
- 你现在可以完全不管 `OCI`、`R2`、`rclone`、`BACKUP_TARGETS`
- 如果你在 macOS 或本地开发机上测试 `init`，配置文件默认会落到当前项目目录，不会强制写 `/opt/vpsmagic`

## 第三步：执行一次本地备份

```bash
cd /opt/vpsmagic
bash vpsmagic.sh backup --config /opt/vpsmagic/config.env
```

正常情况下，归档会出现在：

```bash
/opt/vpsmagic/backups/archives/
```

目录中至少会有两类文件：

- `*.tar.gz`
- `*.tar.gz.sha256`

如果没有看到这两类文件，说明这轮备份不算成功，不要继续往下走。

这一轮完成后，最起码应该满足：

- 备份命令正常结束
- 摘要里 `错误: 0`
- 本地归档目录里同时存在 `*.tar.gz` 和 `*.tar.gz.sha256`

## 第四步：校验归档

```bash
cd /opt/vpsmagic
ls -lh /opt/vpsmagic/backups/archives
sha256sum -c /opt/vpsmagic/backups/archives/*.sha256
```

如果输出为 `OK`，说明本地归档和校验文件匹配。

如果你看到的是 `FAILED`，先不要尝试恢复，优先排查本地磁盘空间或归档是否完整。

如果命令提示找不到 `.sha256` 文件，先回到上一步确认备份是不是实际成功写出了校验文件。

## 第五步：做一次本地恢复演练

建议不要等到出故障才第一次恢复。先在目标环境或测试机上演练：

```bash
bash vpsmagic.sh restore --local /path/to/backup.tar.gz --config /opt/vpsmagic/config.env
```

这一步的目的不是立刻上线，而是确认：

- 归档可读
- 校验可通过
- 恢复流程能走通
- 摘要和健康检查符合预期

建议你至少确认恢复摘要里：

- `错误: 0`
- 关键服务没有被误标成成功
- 如果你用的是 Compose，容器数量和端口检查结果合理

如果你是第一次恢复，建议把目标先放低：

- 先确认恢复流程走通
- 再确认关键服务起来
- 最后再确认业务访问正常

不要一上来就把“网站完全恢复上线”当成唯一判断标准，否则排障会发散得太快。

## 第六步：什么时候再配置远端

等你已经完成一次“本地备份 + 本地恢复演练”后，再进入下一阶段：

1. 安装并配置 `rclone`
2. 配置一个远端目标
3. 测试远端备份
4. 测试远端恢复

下一步建议阅读：

- [配置说明](/Users/terry/Project/Codex/VPSMagicBackup/docs/zh/配置说明.md)
- [备份说明](/Users/terry/Project/Codex/VPSMagicBackup/docs/zh/备份说明.md)
- [恢复说明](/Users/terry/Project/Codex/VPSMagicBackup/docs/zh/恢复说明.md)

一个实用原则：

- 先把“本地备份 + 本地恢复演练”做成功
- 再做“云端备份 + 远端恢复”
- 最后才做“跨机恢复 / 在线迁移”

## 常见误区

### 误区 1：先研究远端，再决定要不要本地备份

不建议。第一次使用时，本地备份成功一次比“先配云端”更重要。

### 误区 2：有了归档就等于可以恢复

不对。至少还要确认：

- `.sha256` 校验通过
- 你知道恢复命令怎么跑
- 你做过一次恢复演练

### 误区 3：第一次恢复就直接上生产

不建议。最稳的顺序仍然是：

1. 先在测试环境演练
2. 再在目标环境恢复
3. 最后才做切换或上线

## 你现在可以忽略的内容

如果你只是第一次上手，目前可以先不管这些高级能力：

- `rclone`
- `BACKUP_TARGETS`
- `BACKUP_PRIMARY_TARGET`
- `BACKUP_ASYNC_TARGET`
- `OCI`
- `R2`
- `--source-hostname`
- `--rollback-on-failure`

先跑通最小闭环，再升级能力，成功率最高。
