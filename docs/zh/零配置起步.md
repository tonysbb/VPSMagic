# 零配置起步

这篇文档面向第一次使用本工具的用户，假设你当前：

- 还没有 `rclone`
- 没有 OCI / R2 / WebDAV 等远端存储
- 只想先成功做出一份可恢复的本地备份

目标很简单：

1. 在当前 VPS 上生成本地归档
2. 校验归档
3. 知道如何用这个归档做本地恢复

## 第一步：安装

```bash
git clone https://github.com/tonysbb/VPSMagic.git /opt/vpsmagic
cd /opt/vpsmagic
bash install.sh
```

## 第二步：创建最小配置

```bash
cp /opt/vpsmagic/config.example.env /opt/vpsmagic/config.env
vim /opt/vpsmagic/config.env
```

至少确认这两项：

```bash
BACKUP_DESTINATION="local"
BACKUP_ROOT="/opt/vpsmagic/backups"
```

说明：

- `local` 表示本次只做本地备份，不依赖远端存储
- 你现在可以完全不管 `OCI`、`R2`、`rclone`、`BACKUP_TARGETS`

## 第三步：执行一次本地备份

```bash
cd /opt/vpsmagic
bash vpsmagic.sh backup --config /opt/vpsmagic/config.env
```

正常情况下，归档会出现在：

```bash
/opt/vpsmagic/backups/archives/
```

目录中至少会有两类文件：

- `*.tar.gz`
- `*.tar.gz.sha256`

## 第四步：校验归档

```bash
cd /opt/vpsmagic
ls -lh /opt/vpsmagic/backups/archives
sha256sum -c /opt/vpsmagic/backups/archives/*.sha256
```

如果输出为 `OK`，说明本地归档和校验文件匹配。

## 第五步：做一次本地恢复演练

建议不要等到出故障才第一次恢复。先在目标环境或测试机上演练：

```bash
bash vpsmagic.sh restore --local /path/to/backup.tar.gz --config /opt/vpsmagic/config.env
```

这一步的目的不是立刻上线，而是确认：

- 归档可读
- 校验可通过
- 恢复流程能走通
- 摘要和健康检查符合预期

## 第六步：什么时候再配置远端

等你已经完成一次“本地备份 + 本地恢复演练”后，再进入下一阶段：

1. 安装并配置 `rclone`
2. 配置一个远端目标
3. 测试远端备份
4. 测试远端恢复

## 你现在可以忽略的内容

如果你只是第一次上手，目前可以先不管这些高级能力：

- `rclone`
- `BACKUP_TARGETS`
- `BACKUP_PRIMARY_TARGET`
- `BACKUP_ASYNC_TARGET`
- `OCI`
- `R2`
- `--source-hostname`
- `--rollback-on-failure`

先跑通最小闭环，再升级能力，成功率最高。
