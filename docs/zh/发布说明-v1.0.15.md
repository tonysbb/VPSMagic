# v1.0.15 发布说明

发布日期：2026-03-30

## 概览

`v1.0.15` 主要收口了远端恢复链路中的可观测性、依赖自愈和误报问题，并补齐了结构化输出与文档入口。

这一版本已经在真实环境中完成以下验证：

- 源机执行远端备份成功
- 目标机从 `R2` 备用远端恢复成功
- 目标机从 `OOS` 主远端恢复成功
- `doctor --format json` 与 `status --format json` 可被真实机解析使用
- 英文与中文主链路文档同步可用

## 重点新增

### 1. `doctor --format json`

新增 `doctor` 的结构化输出，核心字段包括：

- `profile`
- `workload`
- `dependencies`
- `risk_assessment`
- `recommended_path`

这使得恢复前风险评估结果可以直接被自动化脚本、CI 或外部工具消费。

### 2. `status --format json`

新增 `status` 的结构化输出，覆盖：

- 系统信息
- 当前配置
- 启用模块
- 本地备份概览
- 远端备份概览
- 调度状态

## 重点修复

### 1. 远端恢复前置检查收紧

恢复前置检查现在会明确区分：

- 远端未配置
- 远端可配置但当前不可访问
- 主远端不可用但备用远端可用

不再把远端不可达误判成“备份不存在”。

### 2. `rclone` 自愈与远端 backend 识别修复

这一版本连续修正了恢复场景中的 `rclone` 问题：

- 缺少 `rclone` 时，恢复流程可自动安装
- 缺少所需 backend 时，会优先尝试安装官方 release
- 关键恢复路径优先使用 `/usr/local/bin/rclone`
- backend 支持性判断改为基于 `rclone config show <remote>`

最后这一点修复了一个真实误报：

- 旧逻辑会把 `OOS/oracleobjectstorage` 误判为“backend 不支持”
- 新逻辑改为按 remote 配置解析 `type`，已在真实目标机上验证通过

### 3. `init` 交互修整

- 修复远端类型选择后的多余空行
- Telegram 通知在 `Bot Token` 或 `Chat ID` 留空时，不再错误保持启用状态，而是明确保持禁用

### 4. `status` 远端状态更稳

- 某个远端不可读时，文本 `status` 不再整体中断
- JSON `status` 会明确给出每个远端的 `available: true/false`

### 5. Caddy TLS 自愈

恢复流程在检测到特定的 Caddy 本机 TLS 状态异常时，会执行一次受条件约束的本机状态重建，覆盖真实切换中遇到的以下问题：

- `Unable to validate JWS`
- `caddy_legacy_user_removed`

## 文档与产品化整理

这一版本还完成了以下文档与体验整理：

- 修复仓库内文档链接，改为 GitHub 可点击相对路径
- 增加“新用户新环境开局指导”
- 增加英文版最终验收结论
- 增加 `status --format json` 文档
- 整理文档入口结构
- 去除仓库文档中的环境指纹示例

## 实机验收结论

`v1.0.15` 已在真实环境中完成如下闭环：

1. 源机执行备份并上传到主远端与备用远端
2. 目标机空机初始化配置
3. `doctor` 正确识别接入画像与风险
4. `restore` 能根据前置条件选择可用远端
5. 目标机可从 `R2` 完成恢复
6. 目标机可从 `OOS` 完成恢复
7. 最终健康检查与执行摘要均为 `Warnings: 0 / Errors: 0`

## 升级建议

如果你正在从旧版本升级到 `v1.0.15`，建议优先执行：

```bash
cd /opt/vpsmagic
git pull --ff-only origin main
bash vpsmagic.sh doctor --config /opt/vpsmagic/config.env
bash vpsmagic.sh status --format json
```

如果目标机需要执行远端恢复，建议先确认：

- `rclone config show <remote>` 能正确解析 remote
- `doctor` 没有显示阻塞项
- `restore` 前置检查中，主远端与备用远端的可用性与预期一致

## 向后兼容性

本版本未引入破坏性配置键变更。

现有 `config.env` 可继续使用；如需利用新的引导说明与示例，可参考最新的 [config.example.env](../../config.example.env)。
