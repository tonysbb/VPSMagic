# 配置说明

最关键的配置项如下。

## 推荐配置方式

普通用户建议优先使用：

```bash
bash vpsmagic.sh init
```

向导目前支持：

- 仅本地备份
- 本地 + 云端备份
- 仅生成配置，稍后再完善

只有在你明确选择云端模式时，向导才会继续询问远端相关配置。

## 远端目标

```bash
BACKUP_TARGETS="OOS:bucket/vpsmagic/{hostname},R2:mybucket/vpsmagic/{hostname}"
BACKUP_PRIMARY_TARGET="OOS:bucket/vpsmagic/{hostname}"
BACKUP_ASYNC_TARGET="R2:mybucket/vpsmagic/{hostname}"
BACKUP_INTERACTIVE_TARGETS=true
```

说明：

- `BACKUP_TARGETS`：候选远端列表
- `BACKUP_PRIMARY_TARGET`：默认主目标
- `BACKUP_ASYNC_TARGET`：异步副本目标
- `BACKUP_INTERACTIVE_TARGETS`：是否交互列出候选远端

### 当前向导中的远端类型映射

`init` 里的“本地 + 云端备份”分支会先让用户选择远端类型：

1. `WebDAV / OpenList / AList`
2. `S3 兼容对象存储`
3. `Google Drive / OneDrive`
4. `我已经有完整 rclone 路径，直接输入`

这 4 类的默认示例分别是：

```bash
openlist_webdav:backup/{hostname}
s3:mybucket/vpsmagic/{hostname}
gdrive:VPSMagicBackup/{hostname}
manualremote:backup/{hostname}
```

说明：

- `OCI`、`R2` 归类在“S3 兼容对象存储”这一类
- 高级用户仍然可以直接输入完整 `remote:path`

## 备份保留

```bash
BACKUP_KEEP_LOCAL=3
BACKUP_KEEP_REMOTE=30
BACKUP_ROOT="/opt/vpsmagic/backups"
BACKUP_DESTINATION="remote"
```

如果是新手本地模式，向导会自动生成：

```bash
BACKUP_DESTINATION="local"
```

## 恢复相关

```bash
RESTORE_ROLLBACK_ON_FAILURE=false
# RESTORE_SOURCE_HOSTNAME="NCPDE"
```

- `RESTORE_ROLLBACK_ON_FAILURE`：仅控制是否允许通过命令参数启用自动配置级回滚
- `RESTORE_SOURCE_HOSTNAME`：跨机恢复时用于展开 `{hostname}`

## 模块开关

可按需关闭不想备份的模块，例如：

```bash
ENABLE_DATABASE=false
ENABLE_SSL_CERTS=false
```
