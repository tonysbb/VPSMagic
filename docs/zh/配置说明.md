# 配置说明

最关键的配置项如下。

## 推荐配置方式

普通用户建议优先使用：

```bash
bash vpsmagic.sh init
```

向导目前支持：

- 仅本地备份
- 本地 + 云端备份
- 仅生成配置，稍后再完善

只有在你明确选择云端模式时，向导才会继续询问远端相关配置。

## 新手先改哪些，先别改哪些

如果你只是第一次使用，通常只需要先确认：

- `BACKUP_DESTINATION`
- `BACKUP_ROOT`
- `BACKUP_KEEP_LOCAL`

下面这些高级项可以后面再配：

- `BACKUP_TARGETS`
- `BACKUP_PRIMARY_TARGET`
- `BACKUP_ASYNC_TARGET`
- `RESTORE_SOURCE_HOSTNAME`
- `RESTORE_ROLLBACK_ON_FAILURE`

## 三种常见配置层级

### 1. 仅本地模式

最适合第一次使用：

```bash
BACKUP_DESTINATION="local"
BACKUP_TARGETS=""
BACKUP_PRIMARY_TARGET=""
BACKUP_ASYNC_TARGET=""
```

### 2. 单远端模式

最适合已经有一个稳定远端的用户：

```bash
BACKUP_DESTINATION="remote"
BACKUP_TARGETS="openlist_webdav:backup/{hostname}"
BACKUP_PRIMARY_TARGET="openlist_webdav:backup/{hostname}"
BACKUP_ASYNC_TARGET=""
```

### 3. 双远端模式

最适合已经明确需要主备的用户：

```bash
BACKUP_DESTINATION="remote"
BACKUP_TARGETS="OOS:bucket/vpsmagic/{hostname},R2:mybucket/vpsmagic/{hostname}"
BACKUP_PRIMARY_TARGET="OOS:bucket/vpsmagic/{hostname}"
BACKUP_ASYNC_TARGET="R2:mybucket/vpsmagic/{hostname}"
```

## 远端目标

```bash
BACKUP_TARGETS="OOS:bucket/vpsmagic/{hostname},R2:mybucket/vpsmagic/{hostname}"
BACKUP_PRIMARY_TARGET="OOS:bucket/vpsmagic/{hostname}"
BACKUP_ASYNC_TARGET="R2:mybucket/vpsmagic/{hostname}"
BACKUP_INTERACTIVE_TARGETS=true
```

说明：

- `BACKUP_TARGETS`：候选远端列表
- `BACKUP_PRIMARY_TARGET`：默认主目标
- `BACKUP_ASYNC_TARGET`：异步副本目标
- `BACKUP_INTERACTIVE_TARGETS`：是否交互列出候选远端

### 当前向导中的远端类型映射

`init` 里的“本地 + 云端备份”分支会先让用户选择远端类型：

1. `WebDAV / OpenList / AList`
2. `S3 兼容对象存储`
3. `Google Drive / OneDrive`
4. `我已经有完整 rclone 路径，直接输入`

这 4 类的默认示例分别是：

```bash
openlist_webdav:backup/{hostname}
s3:mybucket/vpsmagic/{hostname}
gdrive:VPSMagicBackup/{hostname}
manualremote:backup/{hostname}
```

说明：

- `OCI`、`R2` 归类在“S3 兼容对象存储”这一类
- 高级用户仍然可以直接输入完整 `remote:path`

### 选型建议

- 如果你只是先想能用：选 `WebDAV / OpenList / AList`
- 如果你已经在用对象存储：选 `S3 兼容对象存储`
- 如果你不想折腾对象存储：选 `Google Drive / OneDrive`
- 如果你已经非常清楚自己的 remote:path：选手动输入

如果你没有明确的对象存储方案，建议不要一开始就用双远端模式。

## 备份保留

```bash
BACKUP_KEEP_LOCAL=3
BACKUP_KEEP_REMOTE=30
BACKUP_ROOT="/opt/vpsmagic/backups"
BACKUP_DESTINATION="remote"
```

如果是新手本地模式，向导会自动生成：

```bash
BACKUP_DESTINATION="local"
```

这意味着：

- 现在这份配置不依赖 `rclone`
- 也不会因为没配 `OCI` / `R2` 而阻塞本地备份

## 恢复相关

```bash
RESTORE_ROLLBACK_ON_FAILURE=false
# RESTORE_SOURCE_HOSTNAME="SOURCE_HOST"
```

- `RESTORE_ROLLBACK_ON_FAILURE`：仅控制是否允许通过命令参数启用自动配置级回滚
- `RESTORE_SOURCE_HOSTNAME`：跨机恢复时用于展开 `{hostname}`

### `RESTORE_SOURCE_HOSTNAME` 什么时候用

只在这种场景下需要：

- 你的远端路径写的是 `{hostname}`
- 你在另一台机器上恢复源机备份

例如源机主机名是 `SOURCE_HOST`，目标机执行：

```bash
bash vpsmagic.sh restore \
  --config /opt/vpsmagic/config.env \
  --source-hostname SOURCE_HOST
```

如果你还没有跨机恢复需求，可以先完全忽略这一项。

## 模块开关

可按需关闭不想备份的模块，例如：

```bash
ENABLE_DATABASE=false
ENABLE_SSL_CERTS=false
```
