# 备份说明

## 基本用法

```bash
vpsmagic backup
```

行为：

1. 调用 10 个采集器，扫描和收集本机状态
2. 打包为 `.tar.gz` 归档
3. 生成 `.sha256` 校验文件
4. 如启用远端备份，上传到主远端
5. 如配置异步副本，再复制一份到副本远端

## 备份范围

`vpsmagic backup` 会自动扫描以下 10 个模块：

| 模块 | 采集内容 | 配置开关 |
|------|---------|---------|
| Docker Compose | `docker-compose.yml`、`.env`、Dockerfile、命名卷数据、bind mount 目录、配置文件（yaml/conf/sh/py/json 等）、目录权限快照 | `ENABLE_DOCKER_COMPOSE` |
| 独立 Docker 容器 | 容器配置、挂载卷信息（以重建线索为主） | `ENABLE_DOCKER_STANDALONE` |
| Systemd 服务 | `.service` 文件、`EnvironmentFile`、Python venv 及 `pip freeze` | `ENABLE_SYSTEMD` |
| 反向代理 | Caddy / Nginx / Apache / Traefik 配置 | `ENABLE_REVERSE_PROXY` |
| 数据库 | MySQL / PostgreSQL 逻辑导出、SQLite 文件复制 | `ENABLE_DATABASE` |
| SSL 证书 | Let's Encrypt / acme.sh 证书和账户数据 | `ENABLE_SSL_CERTS` |
| Crontab | 所有用户的 crontab 条目 | `ENABLE_CRONTAB` |
| 防火墙 | UFW / iptables / nftables / firewalld 规则 | `ENABLE_FIREWALL` |
| 用户目录 | `.bashrc`、`.ssh/authorized_keys`、`.config/rclone` 等 | `ENABLE_USER_HOME` |
| 自定义路径 | `EXTRA_PATHS` 中指定的任意文件或目录 | `ENABLE_CUSTOM_PATHS` |

大多数模块支持 `auto` 自动探测，也可以在 `config.env` 中手动指定。

## 备份产出物

正常完成后，归档目录（默认 `/opt/vpsmagic/backups/archives/`）会包含：

```
vpsmagic_<hostname>_<yyyymmdd>_<hhmmss>.tar.gz        # 备份归档
vpsmagic_<hostname>_<yyyymmdd>_<hhmmss>.tar.gz.sha256  # SHA256 校验文件
```

归档内部结构：

```
<backup_name>/
├── manifest.txt        # 备份元信息（主机名、时间、系统版本、VPSMagic 版本）
├── system_info/        # 已安装软件包列表（dpkg/apt/yum/pip/npm/docker images）
├── docker_compose/     # Compose 项目文件、卷数据、bind mounts、目录权限
├── systemd/            # Systemd 服务文件、工作目录、venv 信息
├── reverse_proxy/      # 反代配置文件
├── database/           # 数据库导出
├── ssl_certs/          # SSL 证书
├── crontab/            # Crontab 条目
├── firewall/           # 防火墙规则
├── user_home/          # 用户目录配置
└── custom_paths/       # 自定义路径
```

## 备份模式

### 仅本地备份

```bash
vpsmagic backup --dest local
```

归档只保留在本机。适合：
- 第一次体验
- 不需要异地灾备
- 后续手动传输到目标机

### 本地 + 远端备份

```bash
vpsmagic backup
```

前提：`config.env` 中已配置 `BACKUP_DESTINATION="remote"` 和远端目标。备份会先生成本地归档，再上传到远端。

### 仅上传

```bash
vpsmagic upload
```

不重新执行备份，仅上传最新的本地归档到远端。适合网络中断后补传。

## 备份保留策略

| 配置项 | 默认值 | 说明 |
|--------|--------|------|
| `BACKUP_KEEP_LOCAL` | `3` | 本地保留最近 N 份备份 |
| `BACKUP_KEEP_REMOTE` | `30` | 远端保留最近 N 份备份 |

超出保留数量的旧备份会在每次备份完成后自动清理。

## 加密备份

在 `config.env` 中设置密码后，备份文件将使用 AES-256-CBC 加密：

```bash
BACKUP_ENCRYPTION_KEY="your-strong-password"
```

恢复时需要提供相同密码。加密后的文件后缀为 `.tar.gz.enc`。

## 模拟运行

```bash
vpsmagic backup --dry-run
```

只做扫描和汇报，不实际打包和上传。用于验证配置是否正确。

## 常见问题

### 备份太慢？

- 检查是否有大量 Docker 卷数据需要打包
- 远端上传速度受网络影响，可通过 `RCLONE_BW_LIMIT` 限速避免影响业务

### 备份文件太大？

- 通过模块开关关闭不需要的模块
- 通过 `EXTRA_PATHS` 精确控制自定义路径，避免打包不必要的大文件

### 远端上传失败？

- 先确认 `rclone` 配置正确：`rclone lsd your_remote:path`
- 检查远端存储空间是否充足
- 详见 [排障说明](./排障说明.md)
