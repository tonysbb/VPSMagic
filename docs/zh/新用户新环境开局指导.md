# 新用户新环境开局指导

这篇文档面向这样一种场景：

- 你是第一次使用 `VPS Magic Backup`
- 你面对的是一台还没建立备份习惯的新环境
- 你现在最需要的不是“看完全部文档”，而是先知道该走哪条路径

这篇文档的目标不是替代所有细节文档，而是帮你先选对开局路线。

## 先判断你属于哪种开局

### 路线 A：先做本地最小闭环

适合：

- 你还没有 `rclone`
- 你还没有云端远端
- 你想先确认这个工具在当前机器上能不能工作

目标：

1. 先做出一份本地归档
2. 先做一次本地恢复演练
3. 确认摘要、校验、健康检查都合理

这是默认推荐路线。

### 路线 B：本地 + 远端一起起步

适合：

- 你已经确定要做异地备份
- 你已经有可用的 `rclone` remote
- 你接受先完成一次远端前置检查，再决定是否正式恢复

目标：

1. 先生成一份本地归档
2. 再把它上传到远端
3. 再测试远端恢复前置检查和远端恢复

### 路线 C：跨机恢复 / 切流准备

适合：

- 你已经有源机
- 你已经有目标机
- 你准备从“备份工具是否可用”进入“业务是否能切过去”

目标：

1. 先在目标机跑 `doctor`
2. 先看阻塞项和注意项
3. 先做远端恢复前置检查
4. 再决定是否正式切换

如果你还没跑通过路线 A，不建议直接从路线 C 起步。

## 无论哪条路线，第一步都一样

先安装并初始化：

```bash
git clone https://github.com/tonysbb/VPSMagic.git /opt/vpsmagic
cd /opt/vpsmagic
bash install.sh
bash vpsmagic.sh init
```

然后立刻运行一次：

```bash
bash vpsmagic.sh doctor --config /opt/vpsmagic/config.env
```

现在 `doctor` 会直接给你：

- 当前建议
- 风险等级
- 阻塞项
- 注意项

如果你要把结果交给自动化工具：

```bash
bash vpsmagic.sh doctor --config /opt/vpsmagic/config.env --format json
```

如果你要看当前机器与备份状态：

```bash
bash vpsmagic.sh status --config /opt/vpsmagic/config.env
bash vpsmagic.sh status --config /opt/vpsmagic/config.env --format json
```

## 推荐的第一轮动作

### 如果你是路线 A

按这个顺序：

1. `init`
2. 选择 `仅本地备份`
3. 跑一次 `backup`
4. 校验本地归档和 `.sha256`
5. 做一次 `restore --local`

继续阅读：

- [零配置起步](./零配置起步.md)
- [备份说明](./备份说明.md)
- [恢复说明](./恢复说明.md)

### 如果你是路线 B

按这个顺序：

1. `init`
2. 选择 `本地 + 云端备份`
3. 配好一个远端
4. 跑一次 `backup`
5. 先看 `status`
6. 再跑一次 `restore` 前置检查

继续阅读：

- [快速开始](./快速开始.md)
- [配置说明](./配置说明.md)
- [备份说明](./备份说明.md)
- [状态说明](./状态说明.md)
- [恢复说明](./恢复说明.md)

### 如果你是路线 C

按这个顺序：

1. 在目标机先跑 `doctor`
2. 看阻塞项是否为 `0`
3. 看 `status` 是否能说明当前远端、本地、调度状态
4. 如果远端路径使用 `{hostname}`，准备好 `--source-hostname`
5. 做一次远端恢复前置检查
6. 再决定是否正式恢复

继续阅读：

- [恢复说明](./恢复说明.md)

- [排障说明](./排障说明.md)

## 两个必须接受的边界

### 1. 凭据不是自动补的

这些是安全验证门槛，不是便利项：

- `rclone.conf`
- `/root/.oci/config`

### 2. 第一次恢复不应该直接拿来上线

先做：

1. 本地恢复演练
2. 远端前置检查
3. 目标机恢复
4. 业务验证

最后才做切流。
