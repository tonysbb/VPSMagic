# v1.0.18 发布说明

发布日期：2026-03-30

## 概览

`v1.0.18` 是一次小版本稳定性收口，重点处理在 `NCPDE` 源机重新走备份验证时暴露出的真实使用问题。

这一版本主要收掉了以下问题：

- `doctor` 前期探测没有阶段提示，看起来像卡住
- `status` 在远端慢或异常时可能长时间无响应
- banner 文本不居中、边框被中文宽字符撑坏
- `status` 的远端备份计数与 `restore` 的真实可见备份不一致

## 重点修复

### 1. `status` 远端查询增加超时保护

`status` 现在对每个远端的查询都带超时控制，不再因为某一个远端慢或异常而让整条命令看起来挂住。

### 2. `status` 直接复用 `restore` 的远端列举逻辑

旧版本里，`status` 自己维护了一套远端备份识别规则，这导致它可能和 `restore` 的真实行为分叉。

`v1.0.18` 改为：

- `status` 直接复用 `restore` 已验证通过的远端归档列举 helper
- 文本 `status` 与 JSON `status` 的远端计数，都会和实际恢复路径看到的备份保持一致

### 3. `doctor` 增加阶段提示

`doctor` 现在会在扫描业务形态和评估依赖/远端条件前输出轻量阶段提示，避免慢探测时给人“脚本卡住”的误判。

### 4. banner 渲染修整

banner 现在改为按显示宽度进行居中和裁剪，不再依赖固定宽度空格。

修复内容包括：

- 标题不居中
- 中文宽字符导致边框变形
- 安装完成 banner 风格不一致

## 实机验证

`v1.0.18` 已在源机重新执行远端备份后完成验证：

- 主目标 `OOS` 备份成功
- 异步副本 `R2` 启动成功
- 文本 `status` 正确显示两个远端各有 `1` 份备份
- JSON `status` 正确返回两个远端的 `backup_count: 1` 与 `available: true`
