# 免责声明

本项目是面向常见 VPS 运维场景的工程化 Bash 工具，不是系统镜像产品，也不是事务级灾难恢复平台。

## 使用前请先确认

1. 你接受“先演练，再上线”的使用方式。
2. 你理解恢复成功的定义是“尽量恢复到可运行状态”，不是“系统所有副作用完全逆转”。
3. 你愿意为远端恢复预置必要凭据，例如 `rclone.conf`、`/root/.oci/config`。
4. 你会对数据库、业务任务、回调、队列等业务侧结果做额外校验。

## 本工具不默认承诺的事项

- 完整系统级回滚
- 卷数据回滚
- 数据库业务一致性回滚
- 第三方外部系统副作用回滚
- 对所有发行版的无差异自动恢复

## 建议的上线策略

1. 在源机先做一次最新备份。
2. 在目标机执行完整恢复演练。
3. 检查摘要、端口、容器、反代和业务可用性。
4. 确认无误后再切换 DNS 或业务入口。
