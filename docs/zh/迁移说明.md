# 迁移说明

在线迁移用于两台 VPS 都在线的场景：从源机直接将备份推送到目标机，并在目标机自动恢复。

## 迁移原理

```
源机                                目标机
  │                                   │
  ├── 1. 检测 SSH 连接                  ←┤
  ├── 2. 推送 vpsmagic 源码并安装       →┤
  ├── 3. 执行本地备份                   │
  ├── 4. rsync/scp 传输归档           →┤
  ├── 5. 远程校验归档                 →┤
  │                                   ├── 6. 自动恢复
  │                                   └── 7. 输出恢复摘要
  └── 8. 显示迁移结果 + 检查清单
```

## 前置条件

| 条件 | 说明 |
|------|------|
| SSH 连通性 | 源机能通过 SSH 访问目标机 |
| root 权限 | 目标机需要 root 权限才能恢复服务 |
| 磁盘空间 | 目标机空闲空间应大于备份归档的 2 倍 |
| rsync（推荐） | 如果安装了 rsync，传输速度更快；否则回退为 scp |
| 网络带宽 | 大归档建议限速，避免影响源机业务 |

目标机 **不需要** 预装 vpsmagic — 迁移时会自动安装。

## 基本用法

```bash
vpsmagic migrate root@new-vps
```

这一条命令会完成：本地备份 → 传输 → 远端安装 → 恢复 → 输出结果。

## 常见参数

```bash
# 指定 SSH 端口
vpsmagic migrate root@new-vps -p 2222

# 限制传输带宽（避免影响源机业务）
vpsmagic migrate root@new-vps --bwlimit 10m

# 指定 SSH 密钥
vpsmagic migrate root@new-vps -i ~/.ssh/id_rsa

# 组合使用
vpsmagic migrate root@new-vps -p 2222 --bwlimit 10m

# 只传输备份，不在目标机执行恢复
vpsmagic migrate root@new-vps --skip-restore
```

## 完整迁移流程

### 1. 迁移前检查

```bash
# 确认源机业务正常
vpsmagic status

# 确认 SSH 可达
ssh root@new-vps "echo ok"

# 确认目标机磁盘空间
ssh root@new-vps "df -h /"
```

### 2. 执行迁移

```bash
vpsmagic migrate root@new-vps
```

过程中会显示：
- 备份进度
- 传输进度
- 远端安装状态
- 恢复进度和结果

### 3. 迁移后验证

在目标机上检查：

```bash
# 检查服务状态
vpsmagic status

# 检查 Docker 容器
docker ps

# 检查反代
systemctl status caddy  # 或 nginx

# 检查端口
ss -tlnp | grep -E '80|443'
```

### 4. 切换流量

确认目标机业务正常后：

1. 修改 DNS 记录指向目标机 IP
2. 等待 DNS 生效
3. 确认通过新 IP 可正常访问
4. 源机业务下线（可选）

## 迁移 vs 远端恢复

| 对比项 | 在线迁移 | 远端恢复 |
|--------|---------|---------|
| 前提 | 两台机器都在线，SSH 可达 | 目标机已有云端访问权限 |
| 传输方式 | 源机直推（rsync/scp） | 从云端拉取 |
| 是否需要 rclone | 不需要 | 需要 |
| 适合场景 | 换机、即时迁移 | 灾备恢复、异地重建 |
| 一条命令 | ✅ | ✅ |

## 注意事项

- 迁移过程中源机业务不会停止，但建议在低峰时段操作
- 大量数据传输可能占用带宽，建议使用 `--bwlimit` 限速
- 迁移不会修改 DNS 或切换流量，需要手动操作
- 如果目标机已有同名服务，恢复前会创建配置级快照
- 迁移完成后，建议在目标机重新运行 `vpsmagic init` 配置云端备份
