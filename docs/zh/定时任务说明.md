# 定时任务说明

设置定时备份后，VPS 会自动按计划执行备份，无需人工干预。

## 安装定时任务

```bash
vpsmagic schedule install
```

默认会安装一个 cron 任务，每天凌晨 3:00 自动执行备份。

## 查看状态

```bash
vpsmagic schedule status
```

## 移除定时任务

```bash
vpsmagic schedule remove
```

## 自定义备份时间

在 `config.env` 中修改 `SCHEDULE_CRON`：

```bash
# 默认：每天凌晨 3 点
SCHEDULE_CRON="0 3 * * *"
```

常用示例：

| cron 表达式 | 说明 |
|------------|------|
| `0 3 * * *` | 每天凌晨 3:00 |
| `0 */6 * * *` | 每 6 小时 |
| `0 3 * * 0` | 每周日凌晨 3:00 |
| `0 3 1,15 * *` | 每月 1 号和 15 号凌晨 3:00 |
| `30 2 * * *` | 每天凌晨 2:30 |

修改后需重新安装：

```bash
vpsmagic schedule remove
vpsmagic schedule install
```

## Telegram 通知

建议为定时备份配置通知，避免失败后无人知晓。

在 `config.env` 中配置：

```bash
NOTIFY_ENABLED=true
TG_BOT_TOKEN="123456:ABC-DEF..."
TG_CHAT_ID="你的 Chat ID"
```

获取方法：

1. 在 Telegram 中找 **@BotFather** → 创建 Bot → 获得 Token
2. 在 Telegram 中找 **@userinfobot** → 获得你的 Chat ID

配置后，每次备份完成（成功或失败）都会收到 Telegram 消息。

## 日志查看

定时备份的日志会写入 `/var/log/vpsmagic_cron.log`：

```bash
# 查看最近的备份日志
tail -100 /var/log/vpsmagic_cron.log

# 实时查看日志
tail -f /var/log/vpsmagic_cron.log

# 查看是否有错误
grep -i error /var/log/vpsmagic_cron.log | tail -20
```

## 使用建议

1. **先手动跑通**：在安装定时任务前，至少手动成功执行一次 `vpsmagic backup`
2. **配置通知**：务必开启 Telegram 通知，定时任务出错时能第一时间知道
3. **检查磁盘空间**：定时备份会产生本地归档，注意 `BACKUP_KEEP_LOCAL` 的设置
4. **定期验证**：建议每月至少做一次恢复演练，确认备份可用

## 常见问题

### 定时备份没有执行？

```bash
# 确认 cron 服务运行中
systemctl status cron

# 检查 crontab 是否安装成功
crontab -l | grep vpsmagic

# 查看 cron 日志
grep CRON /var/log/syslog | tail -20
```

### 定时备份执行了但没有通知？

- 确认 `NOTIFY_ENABLED=true`
- 确认 `TG_BOT_TOKEN` 和 `TG_CHAT_ID` 填写正确
- 手动测试通知：在 Telegram 中给 Bot 发一条消息，确认 Bot 能回复
